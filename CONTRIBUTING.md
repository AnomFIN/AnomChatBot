# Contributing to AnomChatBot

Thank you for your interest in contributing to AnomChatBot!

## Code of Conduct

- Be respectful and professional
- Focus on production-quality code
- Maintain existing architecture patterns
- Document your changes

## Development Guidelines

### Code Style

- Use ES6+ modern JavaScript
- Follow existing code patterns
- Add JSDoc comments for functions
- Use meaningful variable names
- Keep functions focused and small

### Architecture Principles

1. **Modularity**: Keep components independent
2. **Error Handling**: Always handle errors gracefully
3. **Logging**: Log important events and errors
4. **Security**: Never commit sensitive data
5. **Testing**: Test your changes thoroughly

### Making Changes

1. **Fork the repository**
2. **Create a feature branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```
3. **Make your changes**
4. **Test thoroughly**
5. **Commit with clear messages**
   ```bash
   git commit -m "Add feature: description"
   ```
6. **Push to your fork**
7. **Create a Pull Request**

### Pull Request Guidelines

- Describe what your PR does
- Reference any related issues
- Include testing steps
- Update documentation if needed
- Keep changes focused and minimal

### Testing Checklist

Before submitting:

- [ ] Code runs without errors
- [ ] All existing features still work
- [ ] New features are documented
- [ ] Error handling is implemented
- [ ] Logging is appropriate
- [ ] No sensitive data in code
- [ ] README updated if needed

## Areas for Contribution

- Bug fixes
- Performance improvements
- Documentation improvements
- New features (discuss first in issues)
- Test coverage
- Error handling improvements

## Questions?

Open an issue for discussion before making large changes.

Thank you for contributing!
